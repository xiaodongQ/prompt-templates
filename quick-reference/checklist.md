# ✅ 提示词检查清单

> 提问前花 30 秒检查，回答质量提升 80%

---

## 🎯 核心五问 (必查)

```
□ 1. 我说明了**背景**吗？(AI 需要知道上下文)
□ 2. 我定义了**任务**吗？(要 AI 做什么)
□ 3. 我指定了**格式**吗？(输出什么样)
□ 4. 我给出了**约束**吗？(长度/风格/限制)
□ 5. 我提供了**示例**吗？(如有，效果更好)
```

---

## 📋 完整清单 (重要任务用)

### 任务定义
```
□ 角色设定：AI 应该扮演什么角色？
□ 任务描述：用一句话清晰说明要做什么
□ 目标受众：内容是给谁看的？
□ 成功标准：什么样的回答算好？
```

### 背景信息
```
□ 项目/任务背景
□ 当前状态/已有进展
□ 相关约束 (时间/预算/技术)
□ 已尝试的方法及结果
□ 相关文档/代码/数据
```

### 输出要求
```
□ 格式：表格/列表/代码/文章/JSON...
□ 长度：字数/页数/条目数
□ 风格：正式/ casual/技术/通俗
□ 结构：需要包含哪些部分
□ 语言：中文/英文/其他
```

### 质量保障
```
□ 提供了正面示例 (应该什么样)
□ 提供了反面示例 (不要什么样)
□ 指定了验证方法 (如何检查质量)
□ 预留了迭代空间 (可以追问)
```

## ⚡ 快速诊断 (回答不好时)

### 问题排查
```
回答太泛？  → 增加具体约束和背景
回答跑题？  → 重新明确任务目标
回答太短？  → 指定长度要求
回答太乱？  → 指定输出格式
回答错误？  → 提供更多信息/示例
```

### 修复模板
```
"我重新说明一下我的需求：

**背景**：[补充之前漏掉的]
**任务**：[更精确的描述]
**要求**：[具体约束]
**格式**：[明确格式]

请基于以上重新回答。"
```

## 🎨 场景速查

### 写代码
```
角色 + 任务 + 语言/版本 + 规范 + 错误处理 + 测试 + 示例
```

### 写文章
```
主题 + 读者 + 目的 + 风格 + 长度 + 结构 + 要点
```

### 分析问题
```
背景 + 问题 + 已尝试 + 期望输出 + 分析维度
```

### 学习理解
```
概念 + 我的水平 + 解释方式 + 示例 + 测试
```

### 修改润色
```
原文 + 修改目标 + 保持什么 + 改动什么 + 风格要求
```

## 🔴 常见错误

### ❌ 太模糊
```
"帮我写个脚本"
→ "帮我写一个 Python 脚本，批量重命名当前目录下所有.txt 文件，
   将空格替换为下划线，先预览变更后执行"
```

### ❌ 缺少背景
```
"这个报错怎么修？"
→ "我在用 Python 3.10 的 requests 库调用 API，
   收到 403 错误。代码是...，已尝试..."
```

### ❌ 没有格式
```
"分析一下这个数据"
→ "分析这个销售数据，用表格展示：
   1. 每月销售额
   2. 环比增长
   3. 最高/最低月份
   4. 趋势分析和建议"
```

### ❌ 一次太多
```
"帮我做个完整的电商网站"
→ 拆解为多个小任务，逐个完成
```

## 🟢 好习惯

### 分步进行
```
"我们先做第一步：[任务 1]
完成后我会确认，然后继续下一步。"
```

### 主动追问
```
"在开始之前，你有什么需要我补充的信息吗？"
```

### 设定角色
```
"你是一位 [领域] 专家，拥有 [经验]，
你的特点是 [特点 1,2,3]"
```

### 要求解释
```
"请在回答后说明你的推理过程，
这样我可以理解你的思路。"
```

### 预留迭代
```
"这是第一轮，我会根据回答提供反馈，
然后请你优化。"
```

## 📊 质量自评

回答完成后问自己：

```
□ 这个回答可以直接用吗？
□ 如果有问题，我知道怎么追问吗？
□ 我学到了新东西吗？
□ 下次类似问题我知道怎么问了吗？
```

## 🎯 万能开场

```markdown
你是一位 [角色]，请帮我 [任务]。

**背景**：[相关信息]
**要求**：[具体约束]
**输出**：[期望格式]

在开始前，如果有任何需要我补充的信息，请告诉我。
```
