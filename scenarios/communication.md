# 💬 沟通协作提示词模板

---

## 📢 表达清晰化

### 想法整理
```markdown
**我的原始想法**：
[零散的想法/感受]

**沟通对象**：[谁]

**沟通目的**：
- [ ] 告知信息
- [ ] 寻求支持
- [ ] 表达不满
- [ ] 请求帮助
- [ ] 提出建议

**请帮我**：
1. 梳理核心观点 (1 句话)
2. 组织逻辑结构
3. 选择合适语气
4. 预判对方反应
5. 准备回应策略

请整理成清晰的表达。
```

### 复杂问题简化
```markdown
**复杂问题**：[描述]

**听众背景**：
- 专业程度：[外行/半专业/专业]
- 关注点：[他们关心什么]
- 时间限制：[几分钟]

**请帮我**：
1. 提炼核心 (1 句话)
2. 设计类比 (用熟悉的事物解释)
3. 准备 3 个关键点
4. 预测可能的疑问
5. 准备简短回答

请帮我简化这个表达。
```

## 🤝 困难对话

### 反馈给予
```markdown
**情境**：需要给 [同事/下属/朋友] 反馈

**问题行为**：[具体描述]

**影响**：[造成了什么影响]

**关系**：[上下级/平级/朋友]

**请用非暴力沟通框架**：

1. **观察** (事实，非评价)
   "我注意到..."

2. **感受** (我的感受)
   "我感到..."

3. **需要** (什么需求未被满足)
   "因为我需要/重视..."

4. **请求** (具体可执行的)
   "你愿意...吗？"

请帮我组织这段对话，语气 [温和/直接/平衡]。
```

### 请求帮助
```markdown
**需要帮助的事项**：[描述]

**求助对象**：[谁]

**关系亲疏**：[熟悉/一般/不熟]

**对方可能的顾虑**：
- [顾虑 1]
- [顾虑 2]

**我能提供的**：
- [回报/互惠]

**请用得体方式表达**：
1. 说明背景
2. 具体请求
3. 说明为什么找 TA
4. 表达理解 (可拒绝)
5. 表示感谢

请帮我写这个请求。
```

### 表达不同意见
```markdown
**议题**：[讨论的话题]

**对方观点**：[描述]

**我的观点**：[描述]

**关系重要性**：[高/中/低]

**请用建设性方式表达**：

1. 先肯定对方合理的部分
2. 用"我"陈述表达不同
3. 提供依据/视角
4. 邀请进一步讨论
5. 保持关系优先

请帮我组织语言。
```

## 🎤 演讲展示

### 演讲结构
```markdown
**演讲主题**：[主题]

**时长**：[分钟]

**听众**：[画像]

**目标**：
- [ ] 告知信息
- [ ] 说服行动
- [ ] 激励鼓舞
- [ ] 娱乐放松

**关键信息**：
1. [信息 1]
2. [信息 2]
3. [信息 3]

**请用经典结构**：

**开场** (10%)
- 钩子 (吸引注意)
- 预告 (今天要讲什么)

**主体** (80%)
- 要点 1 + 例子
- 要点 2 + 例子
- 要点 3 + 例子

**结尾** (10%)
- 总结
- 行动号召
- 金句收尾

请帮我设计演讲大纲。
```

### PPT 内容
```markdown
**演示主题**：[主题]

**页数限制**：[页数]

**每页内容要求**：
- 标题
- 核心要点 (3-5 个 bullet)
- 视觉建议 (图表/图片)

**请为以下结构设计内容**：
1. 封面
2. 议程
3. 背景/问题
4. 方案/内容 (多页)
5. 案例/数据
6. 总结
7. Q&A
8. 结束页

请输出每页的具体内容。
```

## 🤲 谈判协商

### 模板
```markdown
**谈判主题**：[如"薪资"/"合作条款"/"项目范围"]

**我的目标**：
- 理想结果：[最好情况]
- 可接受结果：[底线]
- walk away 点：[无法接受的条件]

**对方可能的目标**：
- [推测 1]
- [推测 2]

**我的筹码**：
- [优势 1]
- [优势 2]

**对方的筹码**：
- [优势 1]
- [优势 2]

**请帮我准备**：
1. 开场策略
2. 论点与依据
3. 可能的反对及回应
4. 让步策略 (什么可以让，什么不能)
5. 备选方案 (BATNA)

请设计谈判策略。
```

## 👥 团队协怍

### 任务分配
```markdown
**项目**：[项目名称]

**团队成员**：
- [成员 A]：[技能/特点]
- [成员 B]：[技能/特点]
- [成员 C]：[技能/特点]

**任务列表**：
- [任务 1]：[要求/难度]
- [任务 2]：[要求/难度]
- [任务 3]：[要求/难度]

**请帮我**：
1. 合理分配任务 (考虑能力与负荷)
2. 明确每个任务的交付标准
3. 设计检查点
4. 准备沟通话术

请生成分配方案。
```

### 进度同步
```markdown
**同步对象**：[上级/团队/客户]

**同步周期**：[日/周/月]

**项目状态**：
- 整体进度：[百分比/描述]
- 本周完成：[列表]
- 下周计划：[列表]
- 风险/问题：[列表]
- 需要支持：[列表]

**请帮我整理**成清晰的同步报告，包含：
1. 执行摘要 (3 句话)
2. 关键进展
3. 风险与应对
4. 需要决策的事项

请生成同步内容。
```

## 🙏 道歉与修复

### 模板
```markdown
**情况**：[发生了什么]

**影响**：[对谁造成了什么影响]

**我的责任**：[承认的部分]

**关系重要性**：[高/中/低]

**请用有效道歉框架**：

1. **明确道歉**："我为...道歉"
2. **承认影响**："我理解这造成了..."
3. **承担责任**："这是我的责任，因为..."
4. **修复方案**："我会..."
5. **预防承诺**："以后我会...避免再次发生"

请帮我组织真诚的道歉。
```

## 🎉 赞美与感谢

### 模板
```markdown
**对象**：[谁]

**原因**：[为什么感谢/赞美]

**关系**：[上下级/同事/朋友]

**场合**：[私下/公开]

**请用具体有效的表达**：

**有效赞美 = 具体行为 + 影响 + 品质**

例如：
"你昨天在会议上提出的方案 (具体行为)，
帮我们节省了至少 2 天的工作量 (影响)，
真的很专业且替团队着想 (品质)。"

请帮我写一个真诚的 [感谢/赞美]。
```
