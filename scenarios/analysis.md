
## 📉 根因分析

### 5 Why 分析法
```markdown
**问题**：[描述发生的问题]

**现象**：[具体表现]

**影响**：[造成的后果]

请用 5 Why 方法分析根因：

1. 为什么发生？→ [答案]
2. 为什么 [答案 1]？→ [答案]
3. 为什么 [答案 2]？→ [答案]
4. 为什么 [答案 3]？→ [答案]
5. 为什么 [答案 4]？→ [根因]

**根本原因**：[总结]

**对策建议**：
- 短期措施
- 长期措施
- 预防机制
```

## 🧩 问题拆解

### 模板
```markdown
**复杂问题**：[描述]

**目标**：将大问题拆解为可执行的小任务

**请使用以下方法拆解**：

1. **第一性原理**：这个问题的本质是什么？
   
2. **MECE 原则**：相互独立、完全穷尽的子问题
   
3. **依赖关系**：哪些任务有先后顺序？
   
4. **优先级**：按影响力和紧急程度排序

**输出格式**：
```
主问题
├── 子问题 1
│   ├── 任务 1.1
│   └── 任务 1.2
├── 子问题 2
│   ├── 任务 2.1
│   └── 任务 2.2
└── 子问题 3
    └── ...
```

请帮我拆解这个问题。
```
