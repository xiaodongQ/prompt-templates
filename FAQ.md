## 🎯 基础问题

### Q1：提示词一定要很长吗？

**不一定。** 长度取决于任务复杂度。

| 任务类型 | 建议长度 | 示例 |
|----------|----------|------|
| 简单查询 | 1-2 句 | "北京明天天气怎么样？" |
| 一般任务 | 3-5 句 | 万能模板 |
| 复杂项目 | 分段详细 | 工作流模式 |

**原则**：让 AI 理解任务所需的最少信息，不要为了长而长。

---

### Q2：AI 回答太泛怎么办？

**原因**：提示词缺少约束。

**修复**：
```
❌ "分析一下这个数据"
✅ "分析这个销售数据，找出：
   1. 销售额最高的 3 个区域
   2. 环比下降最严重的 2 个产品
   3. 用表格呈现，附带 3 条建议"
```

**万能修复句式**：
> "请更具体一些，特别是 [X 方面]。我需要 [具体用途]。"

---

### Q3：AI 理解错了我的意思

**原因**：缺少上下文或有歧义。

**修复**：
1. 补充背景信息
2. 用示例说明
3. 让 AI 复述理解

**示例**：
```
"在开始前，请先复述你理解的任务，确保我们对齐。"
```

---

### Q4：如何让 AI 记住前面的对话？

**方法**：定期总结状态。

```
**当前进度**：完成第 2 步，进入第 3 步
**已确认**：[关键信息 1,2,3]
**本次任务**：[具体任务]
```

**注意**：过长对话 AI 会遗忘，重要信息要重复。

## 🛠️ 进阶问题

### Q5：多轮对话如何保持质量？

**策略**：
1. 每轮开始更新状态
2. 关键节点确认
3. 发现偏差立即纠正

**示例**：
```
"方向对了，但第 2 点需要更深入。请重新展开第 2 点，
补充具体案例和数据支撑。"
```

---

### Q6：如何让 AI 输出特定格式？

**明确指定**：
```
"请用以下 JSON 格式输出：
{
  "summary": "总结",
  "points": ["要点 1", "要点 2"],
  "action": "行动建议"
}"
```

**提供示例**：
```
"参考这个格式：
[示例]

现在处理以下内容：[新内容]"
```

---

### Q7：AI 总是迎合我怎么办？

**明确要求**：
```
"请坦诚评价，不要迎合。指出问题和风险比说好话更有价值。"

"如果有不同意见，请直接说出来。"
```

**设定角色**：
```
"你是一位严格的评审，任务是找出漏洞。"
```

---

### Q8：如何处理复杂任务？

**拆解成工作流**：
```
"这个任务比较复杂，我们分 5 步进行：
1. [步骤 1]
2. [步骤 2]
...

每步完成后我会确认，再继续下一步。

现在开始第 1 步。"
```

## 💡 效率问题

### Q9：写提示词花时间太多，值得吗？

**算笔账**：
- 写好提示词：3 分钟
- 差提示词导致追问：3 轮 × 2 分钟 = 6 分钟
- 还不算心情影响

**结论**：前期多花 1 分钟，节省后面 5 分钟。

**技巧**：把常用提示词保存为模板，越用越快。

## 🔧 技术问题

### Q12：AI 生成的代码有 bug 怎么办？

**正常现象。** AI 不是编译器。

**处理**：
1. 提供完整错误信息
2. 说明运行环境
3. 让 AI 定位问题

**示例**：
```
"代码运行报错：
[错误信息]

环境：Python 3.10, macOS

请定位问题并修复。"
```

---

### Q13：AI 胡说八道（幻觉）怎么办？

**识别**：
- 过于确定的未知信息
- 无法验证的数据/引用
- 违反常识的说法

**预防**：
```
"如果不确定，请明确说明。
对于事实性信息，请标注信心度。
需要验证的信息请提示我查证。"
```

---

### Q14：如何让 AI 更专业？

**设定角色 + 约束**：
```
"你是一位有 15 年经验的 [专家]，服务过 500+ 客户。

你的特点是：
- 专业严谨
- 善于用案例说明
- 会指出潜在风险

请基于这个身份回答。"
```

### Q19：AI 进化了，提示词还重要吗？

**更重要。**

AI 越强，能完成的任务越复杂，越需要清晰的指令。

**趋势**：
- 简单任务：越来越不需要提示词
- 复杂任务：提示词质量决定结果质量

**结论**：基础提示词会简化，高级提示词价值更高。
