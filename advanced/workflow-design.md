# 🔄 AI 工作流设计指南

> 把复杂任务拆解成 AI 可执行的流程

---

## 🎯 什么是 AI 工作流

**定义**：将复杂任务拆解为多个 AI 交互步骤，每步有明确输入输出，最终达成目标。

**vs 单次提示**：
| 维度 | 单次提示 | 工作流 |
|------|----------|--------|
| 适用场景 | 简单任务 | 复杂任务 |
| 对话轮次 | 1-3 轮 | 5-10+ 轮 |
| 质量控制 | 依赖一次输出 | 每步验证 |
| 可复用性 | 低 | 高 |
| 时间投入 | 少 | 多（但 ROI 高） |

---

## 📐 工作流设计原则

### 1. 单一步骤原则

**错误**：一步完成太多事
```
帮我分析市场、设计产品、制定营销策略、写执行计划
```

**正确**：拆成独立步骤
```
第 1 步：市场分析
第 2 步：产品设计
第 3 步：营销策略
第 4 步：执行计划
```

**为什么**：AI 注意力有限，步骤越单一，质量越高。

---

### 2. 明确输入输出

**每个步骤必须有**：
- 输入：上一步的输出 / 用户提供的信息
- 处理：AI 要做什么
- 输出：可验证的交付物

**示例**：
```
步骤 2：产品设计
- 输入：步骤 1 的市场分析报告
- 处理：基于市场机会设计产品功能
- 输出：产品功能列表 + 优先级 + MVP 定义
```

---

### 3. 检查点设计

**在关键节点设置确认**：
```
完成市场分析后，请暂停。

我会确认：
1. 分析方向是否正确
2. 是否需要补充信息
3. 是否继续下一步

得到确认后，再继续产品设计。
```

**为什么**：避免方向偏差累积到最后才发现。

---

### 4. 状态传递

**让 AI 记住上下文**：
```
**当前进度**：完成第 2 步（产品设计），进入第 3 步（营销策略）

**已确认信息**：
- 目标用户：[来自步骤 1]
- 核心功能：[来自步骤 2]
- 差异化定位：[来自步骤 2]

**第 3 步任务**：基于以上信息设计营销策略
```

## 🏗️ 工作流模板

### 模板 1：内容创作工作流

```markdown
**任务**：创作 [内容类型，如"白皮书"/"课程"/"系列文章"]

**工作流设计**：

**阶段 1：规划（第 1-2 次对话）**
- 1.1 目标读者分析
- 1.2 核心信息定义
- 1.3 内容大纲设计
- ✅ 检查点：确认大纲

**阶段 2：创作（第 3-6 次对话）**
- 2.1 引言部分
- 2.2 主体部分 1
- 2.3 主体部分 2
- 2.4 主体部分 3
- ✅ 检查点：完整初稿

**阶段 3：优化（第 7-8 次对话）**
- 3.1 逻辑审查
- 3.2 语言润色
- 3.3 格式美化
- ✅ 检查点：最终版本

**阶段 4：衍生（第 9-10 次对话）**
- 4.1 社交媒体摘要
- 4.2 演示文稿
- 4.3 FAQ 文档

**状态管理**：
每次对话开始时，我会说明当前阶段和步骤。
请基于已确认的内容继续，不要重复已完成的。

现在开始阶段 1.1：目标读者分析。
```

### 模板 2：项目开发工作流

```markdown
**任务**：从 0 开发一个 [项目类型]

**工作流设计**：

**Sprint 0：需求与设计（3 次对话）**
- 0.1 需求收集与分析
  - 输入：我的初步想法
  - 输出：需求文档（用户故事 + 验收标准）
  
- 0.2 技术方案设计
  - 输入：需求文档
  - 输出：技术架构 + 技术选型 + 风险评估
  
- 0.3 项目计划
  - 输入：技术方案
  - 输出：WBS + 时间估算 + 里程碑

**Sprint 1-N：迭代开发（每轮对话一个功能）**
- N.1 功能设计
  - 输入：需求文档中的用户故事
  - 输出：详细设计 + 接口定义
  
- N.2 代码实现
  - 输入：详细设计
  - 输出：完整代码 + 单元测试
  
- N.3 代码审查
  - 输入：代码
  - 输出：审查意见 + 优化建议

**Sprint Final：部署与文档（2 次对话）**
- F.1 部署指南
  - 输出：部署脚本 + 环境配置说明
  
- F.2 用户文档
  - 输出：使用手册 + API 文档

**协作规则**：
1. 每个 Sprint 开始前确认需求
2. 代码完成后必须审查
3. 阻塞问题立即提出
4. 每次对话更新项目状态

现在开始 Sprint 0.1：需求收集与分析。
```

### 模板 3：学习辅导工作流

```markdown
**任务**：系统学习 [主题]

**工作流设计**：

**第 1 次：水平评估与路径设计**
- 评估我的现有知识（5-10 道题）
- 设计个性化学习路径
- 确定学习节奏和资源

**第 2-N 次：主题学习（每轮一个主题）**
- 概念讲解（用类比）
- 核心原理（深度适中）
- 实际案例（2-3 个）
- 小测试（3-5 题）
- 常见误区预警

**第 N+1 次：综合练习**
- 综合案例分析
- 跨主题连接
- 实战项目设计

**第 N+2 次：复习与巩固**
- 知识图谱回顾
- 弱项强化
- 进阶方向建议

**学习规则**：
1. 每次开始前复习上次内容（5 分钟）
2. 根据我的测试表现调整难度
3. 鼓励提问，不理解随时打断
4. 每 3 个主题做一次综合测试

现在开始第 1 次：水平评估。
```

## 🔧 工作流管理技巧

### 1. 状态追踪

**在长工作流中，定期总结状态**：
```
**项目状态更新**（第 5 次对话）

✅ 已完成：
- 需求文档（已确认）
- 技术方案（已确认）
- 用户认证模块（已完成）

🔄 进行中：
- 数据模块（当前步骤）

⏳ 待开始：
- API 模块
- 前端界面
- 测试部署

📋 本次任务：完成数据模块的代码实现
```

---

### 2. 分支处理

**当遇到决策点时**：
```
**决策点**：数据库选择

**选项**：
A. PostgreSQL - 稳定、功能全、学习曲线稍陡
B. MongoDB - 灵活、开发快、一致性弱
C. SQLite - 简单、零配置、不适合高并发

**建议**：基于你的需求（小型项目、快速原型），推荐 C

**请确认**：
- 选择 C，继续开发
- 选择 A/B，我重新设计数据层
- 需要更多对比信息

确认后继续下一步。
```

---

### 3. 回滚机制

**当发现前面有问题时**：
```
**问题**：实现中发现技术方案有缺陷

**影响**：需要调整数据模型设计

**选项**：
1. 小调整：在当前基础上修改（预计 +2 小时）
2. 中调整：重新设计数据层（预计 +1 天）
3. 大调整：回滚到技术方案重新设计（预计 +3 天）

**建议**：选项 1，风险可控

**请确认**：选择哪个方案？

确认后从相应步骤继续。
```

---

### 4. 并行任务

**当多个任务可并行时**：
```
**当前状态**：后端开发完成 50%

**可并行任务**：
- 后端：继续 API 开发（主线程）
- 前端：可开始页面框架（需前端开发者）
- 文档：可开始 API 文档草稿（可交给 AI）

**建议**：
我继续后端开发，同时你可以：
1. 让另一个 AI 实例开始文档
2. 或者我完成后端再处理文档

**选择**：哪个优先级更高？
```

## 📊 工作流质量评估

### 检查清单

设计完工作流后，问自己：

```
□ 每个步骤都有明确输入输出吗？
□ 有关键检查点吗？
□ 状态可追踪吗？
□ 有分支/回滚处理吗？
□ 每步工作量合理吗？（AI 一次能完成）
□ 总轮次合理吗？（不过长）
□ 有最终交付物定义吗？
```

### 常见错误

| 错误 | 表现 | 修复 |
|------|------|------|
| 步骤太大 | AI 回答泛泛而谈 | 继续拆解 |
| 缺少检查点 | 最后发现方向错了 | 增加确认环节 |
| 状态丢失 | AI 忘记前面内容 | 定期总结状态 |
| 没有分支 | 遇到决策卡住 | 预设决策点 |
| 轮次过多 | 10+ 轮还没完成 | 合并相似步骤 |

## 🎯 实战案例

### 案例：用工作流完成市场研究报告

**任务**：为 CEO 准备一份 AI 趋势研究报告

**工作流**：
```
**总览**：8 次对话，5 天完成

**Day 1：范围定义（2 次对话）**
1.1 确定报告范围
   - 输入：CEO 的初步要求
   - 输出：报告边界 + 关键问题列表
   
1.2 信息源规划
   - 输入：关键问题列表
   - 输出：调研计划 + 信息来源

**Day 2-3：信息收集（3 次对话）**
2.1 行业趋势收集
2.2 竞品动态收集
2.3 技术进展收集

**Day 4：分析整合（2 次对话）**
3.1 数据整理与洞察提取
3.2 报告初稿

**Day 5：优化交付（1 次对话）**
4.1 最终润色 + 演示稿

**检查点**：
- Day 1 结束：确认范围
- Day 3 结束：确认洞察方向
- Day 4 结束：确认初稿

**状态追踪**：每次对话开头更新进度
```

**实际效果**：5 天完成，CEO 满意，直接用于战略会议

## 💡 高级技巧

### 1. 多 AI 协作

**场景**：不同 AI 擅长不同任务

```
**工作流**：
- AI-A（分析型）：市场研究、数据分析
- AI-B（创意型）：内容创作、方案设计
- AI-C（审查型）：质量检查、风险识别

**协调**：由你（用户）做信息传递和整合
```

---

### 2. 模板化复用

**把成功的工作流保存为模板**：
```
**模板名称**：月度报告工作流
**适用场景**：每月向管理层汇报
**步骤数**：4 次对话
**总耗时**：约 2 小时
**复用次数**：已用 6 次

[保存完整工作流定义]
```

---

### 3. 持续优化

**每次工作流完成后复盘**：
```
**复盘问题**：
1. 哪一步最耗时？为什么？
2. 哪一步质量最差？如何改进？
3. 有可以合并的步骤吗？
4. 有遗漏的环节吗？
5. 下次如何优化？

**更新模板**：把优化写入工作流定义
```
