# 📈 提示词效果对比库

> 看同样的任务，不同问法带来的巨大差异

---

## 💻 编程场景

### 案例 1：代码生成

#### ❌ 差提示词
```
帮我写个 Python 脚本
```

**AI 回答：**
> 好的，请问您需要什么功能的脚本呢？Python 可以用于很多方面，比如：
> - 数据处理
> - 网络爬虫
> - 自动化任务
> - Web 开发
> ...
> 请告诉我具体需求。

**问题：** 太模糊，AI 无法下手，需要来回追问。

---

#### ✅ 好提示词
```markdown
你是一位资深 Python 工程师。

**任务**：写一个脚本，批量重命名当前目录下所有文件，将文件名中的空格替换为下划线。

**要求**：
- Python 3.10+
- 递归处理子目录
- 先预览变更，用户确认后再执行
- 添加错误处理（权限不足、文件名冲突等）
- 输出操作日志

**输出**：
1. 完整可运行的代码（单文件）
2. 使用说明
3. 3 个测试场景
```

**AI 回答：** 直接输出完整、可运行、专业的代码，包含所有要求的功能。

**效果提升：** ⭐⭐⭐⭐⭐ 一次得到可用结果，节省 3-4 轮对话

### 案例 2：Bug 调试

#### ❌ 差提示词
```
这个报错了，怎么修？
```

**AI 回答：**
> 请问是什么错误？能否提供更多信息：
> - 错误信息
> - 代码
> - 运行环境
> ...

**问题：** 缺少关键信息，无法诊断。

---

#### ✅ 好提示词
```markdown
**问题**：Python requests 调用 API 返回 403 错误

**环境**：
- Python 3.10
- requests 2.31.0
- 目标 API：https://api.example.com/data

**代码**：
```python
import requests
response = requests.get('https://api.example.com/data')
print(response.status_code)  # 输出 403
```

**已尝试**：
- ✅ 检查 URL 正确
- ✅ 网络连通正常
- ❌ 未添加认证头

**预期**：成功获取数据

请分析可能原因和解决方案。
```

**AI 回答：** 精准定位问题（缺少认证/权限），给出具体修复代码和多种可能原因。

**效果提升：** ⭐⭐⭐⭐⭐ 直接得到可执行的解决方案

## ✍️ 写作场景

### 案例 3：文章写作

#### ❌ 差提示词
```
帮我写一篇文章
```

**AI 回答：**
> 好的，请问您想写什么主题的文章？什么风格？多长？给谁看？

**问题：** 完全没有方向。

---

#### ✅ 好提示词
```markdown
你是一位科技自媒体作者，擅长把复杂技术讲得通俗易懂。

**主题**：为什么 2026 年是 AI 应用的爆发年

**目标读者**：30 岁左右的企业中层，对 AI 有兴趣但不太懂技术

**文章目的**：帮助他们理解趋势，做出业务决策

**风格要求**：
- 语气：专业但不晦涩，有洞察力
- 长度：2000 字左右
- 结构：问题引入 → 趋势分析 → 案例说明 → 行动建议

**必须包含**：
- 3 个具体的行业案例
- 2-3 个可操作的建议
- 避免技术术语堆砌

**不要**：
- 空洞的预测
- 过度炒作
```

**AI 回答：** 一篇结构完整、有案例、有洞察的高质量文章。

**效果提升：** ⭐⭐⭐⭐⭐ 直接可用，只需少量修改

**语气**：专业、诚恳、不过度热情

请提供：
1. 3 个邮件主题选项
2. 完整正文
3. 如 3 天无回复的跟进版本
```

**效果提升：** ⭐⭐⭐⭐⭐ 专业得体，提高回复率

## 📊 分析场景

### 案例 5：方案对比

#### ❌ 差提示词
```
A 和 B 哪个更好？
```

**AI 回答：**
> 这取决于您的具体需求。A 和 B 各有优劣...

**问题：** 没有评估维度，无法给出有意义的建议。

---

#### ✅ 好提示词
```markdown
**决策主题**：选择前端框架 - React vs Vue vs Svelte

**项目背景**：
- 类型：企业级 SaaS 仪表盘
- 团队：5 人，都熟悉 JavaScript，无框架深度经验
- 时间：3 个月 MVP

**评估维度**（按重要性排序）：
1. 学习曲线（最重要）
2. 开发效率
3. 长期维护性
4. 性能表现
5. 生态成熟度

**约束条件**：
- 必须支持 TypeScript
- 需要丰富的 UI 组件库
- 考虑 3-5 年的技术生命周期

请用表格对比，并给出明确推荐和理由。
```

**AI 回答：** 结构化对比表格 + 针对具体场景的推荐 + 风险提示。

**效果提升：** ⭐⭐⭐⭐⭐ 可直接用于团队决策讨论

## 📖 学习场景

### 案例 6：概念理解

#### ❌ 差提示词
```
什么是机器学习？
```

**AI 回答：**
> 机器学习是人工智能的一个分支，它使计算机能够从数据中学习...

**问题：** 教科书式定义，看了还是不懂。

---

#### ✅ 好提示词
```markdown
我想理解"机器学习"这个概念。

**我的背景**：
- 熟悉：基本的编程概念
- 不熟悉：统计学、高等数学

**请用费曼技巧解释**：
1. 用最简单的语言说清楚核心是什么
2. 用生活中的类比帮助理解
3. 告诉我它和传统编程的区别
4. 举 2 个实际应用的例子
5. 指出常见的误解

**避免**：数学公式、专业术语堆砌

请像给聪明的 12 岁孩子讲解一样。
```

**AI 回答：** 用类比（如"教小孩认动物"）解释，有例子，有对比，真正理解。

**效果提升：** ⭐⭐⭐⭐⭐ 真正理解而非死记硬背

## ⚡ 效率场景

### 案例 7：任务拆解

#### ❌ 差提示词
```
我想做一个网站，怎么开始？
```

**AI 回答：**
> 做网站需要考虑很多方面，比如需求分析、设计、开发...

**问题：** 太泛，不知道从哪下手。

---

#### ✅ 好提示词
```markdown
**大任务**：为公司做一个产品展示网站

**最终目标**：上线一个能展示产品、联系方式、案例的网站

**截止时间**：4 周后

**可用资源**：
- 时间：我每天 2 小时
- 技术：我会基础 HTML/CSS，不会后端
- 预算：5000 元（可外包部分）

**请帮我拆解**：
1. 分几个阶段（里程碑）
2. 每个阶段的具体任务
3. 哪些可以自己做，哪些建议外包
4. 每周的行动计划
5. 关键风险点

**输出格式**：任务树 + 时间表
```

**AI 回答：** 清晰的 4 周计划，标注了每阶段任务、时间估算、外包建议。

**效果提升：** ⭐⭐⭐⭐⭐ 拿到就能执行的计划

## 💬 沟通场景

### 案例 8：困难对话

#### ❌ 差提示词
```
怎么跟老板说加薪？
```

**AI 回答：**
> 和老板谈加薪需要注意时机、准备充分...

**问题：** 正确的废话，没有可操作性。

---

#### ✅ 好提示词
```markdown
**情境**：想和老板谈加薪

**我的情况**：
- 入职时间：18 个月
- 上次调薪：12 个月前（5%）
- 当前表现：独立负责核心项目，绩效 A
- 目标涨幅：20%

**公司情况**：
- 规模：100 人
- 现状：刚完成融资，业务增长
- 文化：相对开放

**我的顾虑**：
- 怕被拒绝后尴尬
- 不确定时机是否合适
- 不知道具体怎么说

**请帮我准备**：
1. 时机判断（现在是否合适）
2. 谈话框架（开场→陈述→回应）
3. 具体话术（每阶段说什么）
4. 可能的回应及应对
5. 备选方案（如被拒绝）

请用非暴力沟通的原则。
```

**AI 回答：** 完整的谈话脚本，包含具体话术、应对策略、备选方案。

**效果提升：** ⭐⭐⭐⭐⭐ 拿着就能用，降低心理压力

## 🎯 核心差异总结

| 维度 | 差提示词 | 好提示词 |
|------|----------|----------|
| **背景** | 无 | 充分的项目/个人背景 |
| **任务** | 模糊 | 清晰具体的任务描述 |
| **约束** | 无 | 明确的时间/资源/风格限制 |
| **格式** | 无要求 | 指定输出格式和结构 |
| **示例** | 无 | 提供参考或反例 |
| **对话轮次** | 3-5 轮 | 1-2 轮 |
| **结果质量** | 泛泛而谈 | 可直接使用 |

## 💡 关键洞察

**好提示词的本质不是技巧，是思考。**

当你问不出好问题时，通常是因为：
1. 你自己没想清楚要什么
2. 你没有站在 AI 角度想它需要什么信息
3. 你忽略了上下文的重要性

**3 秒自检：**
> "如果我把这个问题发给一个聪明的朋友，TA 能直接给我想要的答案吗？"

如果不能，先补充信息再问 AI。
