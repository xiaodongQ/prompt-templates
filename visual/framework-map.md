## 🧩 CO-STAR 框架详解

```
┌─────────────────────────────────────────────────────────┐
│                    CO-STAR 框架                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  C ── Context ──→ 背景信息                              │
│       "我正在开发一个电商网站..."                         │
│                    ↓                                    │
│  O ── Objective ──→ 任务目标                            │
│       "需要设计用户登录系统的数据库 schema"               │
│                    ↓                                    │
│  S ── Style ──────→ 写作风格                            │
│       "使用专业的技术文档风格"                           │
│                    ↓                                    │
│  T ── Tone ───────→ 语气态度                            │
│       "严谨、准确、避免歧义"                             │
│                    ↓                                    │
│  A ── Audience ───→ 目标受众                            │
│       "读者是有 3 年经验的后端工程师"                     │
│                    ↓                                    │
│  R ── Response ───→ 输出格式                            │
│       "用 Markdown 表格呈现，包含字段名、类型、说明"       │
│                                                         │
└─────────────────────────────────────────────────────────┘

使用频率：★★★★★  (80% 的场景都适用)
```

## 🎭 高级模式使用地图

```
任务类型                      推荐模式
  │
  ├─ 数学/逻辑推理 ──────────→ 思维链 (Chain of Thought)
  │                            "请逐步思考，展示推理过程"
  │
  ├─ 需要专业视角 ───────────→ 角色扮演 (Role Playing)
  │                            "你是一位资深..."
  │
  ├─ 需要机器处理 ───────────→ 结构化输出 (Structured Output)
  │                            "请以 JSON 格式输出..."
  │
  ├─ 复杂项目 ───────────────→ 迭代优化 (Iterative Refinement)
  │                            "我们分 3 轮进行..."
  │
  ├─ 需要模仿风格 ───────────→ 少样本学习 (Few-Shot)
  │                            "参考以下示例..."
  │
  ├─ 决策类问题 ─────────────→ 多视角分析 (Multi-Perspective)
  │                            "从支持者/反对者/中立者角度..."
  │
  ├─ 高质量要求 ─────────────→ 自我反思 (Self-Reflection)
  │                            "请先检查假设和漏洞..."
  │
  ├─ 长文本/复杂任务 ────────→ 分块处理 (Chunking)
  │                            "我们分 4 步进行..."
  │
  ├─ 预测/演练 ──────────────→ 场景模拟 (Scenario Simulation)
  │                            "让我们模拟这个场景..."
  │
  └─ 方案选择 ───────────────→ 评估对比 (Evaluation & Comparison)
                               "请按以下标准评分..."
```

## 📋 快速决策树

```
有新任务
   │
   ↓
任务复杂度？
   │
   ├─ 简单 (1 句话能说清)
   │     │
   │     ↓
   │   用 TAG 框架
   │   "任务 + 动作 + 目标"
   │
   ├─ 中等 (需要一些背景)
   │     │
   │     ↓
   │   用 CO-STAR 框架
   │   填 6 个要素
   │
   └─ 复杂 (多步骤/多轮次)
         │
         ↓
       用工作流模式
       拆解成多轮对话
         │
         ↓
       每轮用 CO-STAR
```
