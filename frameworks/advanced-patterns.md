# 🎓 高级提示词模式

> 掌握这些模式，解决复杂问题

---

## 1. 🧠 思维链 (Chain of Thought)

**适用**：数学、逻辑推理、复杂分析

### 基础版
```markdown
请逐步思考，展示你的推理过程。
Let's think step by step.
```

### 进阶版
```markdown
在给出最终答案前，请：
1. 分析问题中的关键信息
2. 列出可能的解决路径
3. 评估每条路径的优劣
4. 选择最优方案并说明理由
5. 逐步执行并验证结果
```

### 示例
```markdown
问题：一个水池，单独开进水管 6 小时注满，单独开排水管 8 小时排空。
两管同时开，多久注满？

请逐步思考：
1. 先计算每小时的进水量和排水量
2. 计算净进水量
3. 计算注满所需时间
4. 验证答案合理性
```

## 2. 🎭 角色扮演 (Role Playing)

**适用**：需要特定视角或专业知识的任务

### 模板
```markdown
你是一位 [具体角色]，拥有 [相关经验/技能]。

你的特点是：
- [特点 1]
- [特点 2]
- [特点 3]

请用这个角色的视角来 [任务]。
```

### 角色示例库

| 场景 | 角色设定 |
|------|----------|
| 代码审查 | "你是有 10 年经验的技术主管，以严格著称但善于指导" |
| 产品建议 | "你是资深产品经理，擅长从用户角度思考" |
| 写作润色 | "你是《纽约客》的编辑，追求文字精准优雅" |
| 学习辅导 | "你是耐心的大学教授，善于用类比解释复杂概念" |
| 商业分析 | "你是麦肯锡顾问，用结构化思维分析问题" |

### 完整示例
```markdown
你是一位有 15 年经验的安全工程师，曾负责过大型互联网公司的安全架构。

你的特点是：
- 对潜在风险极度敏感
- 善于用通俗语言解释技术问题
- 总是给出可落地的建议

请审查以下代码的安全问题：
[代码]

输出要求：
1. 按严重程度排序问题
2. 每个问题说明风险原理
3. 给出具体的修复方案
```

## 3. 📐 结构化输出 (Structured Output)

**适用**：需要机器处理或标准化格式的任务

### JSON 输出
```markdown
请以 JSON 格式输出，结构如下：
{
  "summary": "简要总结",
  "keyPoints": ["要点 1", "要点 2", "要点 3"],
  "confidence": "高/中/低",
  "recommendations": [
    {"action": "行动项", "priority": "优先级", "effort": "工作量"}
  ]
}
```

### 表格输出
```markdown
请用表格呈现，包含以下列：
| 项目 | 描述 | 优先级 | 预计时间 | 依赖项 |
```

### 分级输出
```markdown
请按以下结构组织回答：

## 📌 核心结论
[一句话总结]

## 🔑 关键要点
- 要点 1
- 要点 2
- 要点 3

## 📖 详细说明
[展开说明]

## ⚠️ 注意事项
[需要警惕的地方]

## 🚀 下一步行动
[具体可执行的建议]
```

## 4. 🔄 迭代优化 (Iterative Refinement)

**适用**：需要多轮改进的任务

### 模板
```markdown
请按照以下步骤进行：

**第一轮**：快速给出初步方案，覆盖主要点即可

**第二轮**：基于以下反馈优化
- [反馈 1]
- [反馈 2]

**第三轮**：最终打磨，关注细节和完整性

请先开始第一轮。
```

### 示例
```markdown
帮我写一个产品发布会的演讲稿。

**第一轮**：先给出大纲和核心信息（5 分钟内看完）

**第二轮**：我会反馈需要强调的卖点和要回避的敏感点

**第三轮**：润色语言，加入故事和金句

现在开始第一轮。
```

## 5. 🎯 少样本学习 (Few-Shot Learning)

**适用**：需要 AI 理解特定风格或模式的任务

### 模板
```markdown
请模仿以下示例的风格/格式：

**示例 1**：
[输入] → [输出]

**示例 2**：
[输入] → [输出]

**示例 3**：
[输入] → [输出]

现在请处理：
[新的输入]
```

### 示例
```markdown
请按照以下格式总结文章：

**示例**：
输入：一篇关于 AI 发展的长文
输出：
📰 标题：AI 发展的下一个十年
⏱️ 阅读时间：3 分钟
💡 核心观点：AI 将从工具演变为合作伙伴
🔑 3 个要点：
   1. 多模态能力成为标配
   2. 垂直领域应用爆发
   3. 人机协作模式成熟

现在请总结这篇文章：
[文章内容]
```

## 6. 🎪 多视角分析 (Multi-Perspective Analysis)

**适用**：需要全面考量的决策类问题

### 模板
```markdown
请从以下角度分析这个问题：

**支持者视角**：为什么这是一个好主意？

**反对者视角**：潜在风险和反对理由是什么？

**中立者视角**：客观的事实和数据是什么？

**执行者视角**：实际落地会遇到什么困难？

**用户视角**：最终使用者的感受和需求是什么？

最后，综合各视角给出你的建议。
```

## 7. 🔍 自我反思 (Self-Reflection)

**适用**：提高回答质量，减少错误

### 模板
```markdown
在给出最终答案前，请先：

1. **检查假设**：我的回答基于哪些假设？这些假设合理吗？

2. **寻找漏洞**：我的推理中有什么逻辑漏洞或遗漏？

3. **考虑边界**：在什么情况下这个答案不适用？

4. **评估信心**：我对这个答案的信心程度（高/中/低）及原因

然后给出最终答案。
```

## 8. 📚 分块处理 (Chunking)

**适用**：处理长文本或复杂任务

### 模板
```markdown
这个任务比较复杂，我们分步进行：

**步骤 1/4**：[第一步任务]
完成后我会确认，然后继续下一步。

**步骤 2/4**：[第二步任务]
...

**步骤 3/4**：[第三步任务]
...

**步骤 4/4**：[第四步任务]
...

现在开始步骤 1/4。
```

## 9. 🎬 场景模拟 (Scenario Simulation)

**适用**：预测结果、演练对话、测试方案

### 模板
```markdown
让我们模拟以下场景：

**背景**：[场景设定]

**角色**：
- 角色 A：[特点]
- 角色 B：[特点]

**情境**：[具体情境]

请模拟对话/事件发展，展示：
1. 可能的对话内容/事件流程
2. 关键转折点
3. 可能的结果及概率
```

## 10. 🏆 评估与对比 (Evaluation & Comparison)

**适用**：方案选择、质量评估

### 模板
```markdown
请评估以下 [方案/作品/答案]，使用以下标准：

**评估维度**：
- 准确性 (0-10 分)
- 完整性 (0-10 分)
- 实用性 (0-10 分)
- 创新性 (0-10 分)

**对每个方案**：
1. 给出各维度分数
2. 说明打分理由
3. 指出最大优势
4. 指出最大不足

**最后**：
- 综合排名
- 推荐选择及理由
- 改进建议
```

## 💡 组合使用示例

```markdown
你是一位资深技术顾问 (角色扮演)

## 背景
公司要在微服务和单体架构之间做选择

## 任务
请从多个视角分析这个问题

## 要求
1. 使用多视角分析框架
2. 每个视角用思维链展示推理
3. 最后用表格对比总结
4. 给出有信心的推荐

## 输出格式
### 各视角分析
...

### 对比表格
| 维度 | 微服务 | 单体 |
...

### 推荐方案
...

### 我的信心度及理由
...
```
