# 🧩 主流提示词框架合集

> 掌握这些框架，让提示词设计有章可循

---

## 1️⃣ CO-STAR 框架 ⭐ 推荐首选

**最全面、最易用的通用框架**

| 要素 | 含义 | 示例 |
|------|------|------|
| **C**ontext | 背景信息 | "我正在开发一个电商网站..." |
| **O**bjective | 任务目标 | "需要设计用户登录系统的数据库 schema" |
| **S**tyle | 写作风格 | "使用专业的技术文档风格" |
| **T**one | 语气态度 | "严谨、准确、避免歧义" |
| **A**udience | 目标受众 | "读者是有 3 年经验的后端工程师" |
| **R**esponse | 输出格式 | "用 Markdown 表格呈现，包含字段名、类型、说明" |

### 模板

```markdown
## Context
[描述任务背景和现状]

## Objective
[明确说明需要完成的任务]

## Style
[指定写作/回答风格]

## Tone
[指定语气态度]

## Audience
[说明内容给谁看]

## Response Format
[指定输出格式：列表/表格/代码/步骤等]
```

### 完整示例

```markdown
## Context
我是一名全栈开发者，正在用 Next.js 14 + TypeScript 构建一个 SaaS 仪表盘。

## Objective
帮我设计用户认证模块的 API 路由结构，包括注册、登录、登出、刷新 token。

## Style
技术文档风格，清晰简洁

## Tone
专业、准确

## Audience
团队中的中级开发者，熟悉 REST API 但需要明确规范

## Response Format
- 用表格列出每个端点
- 包含：方法、路径、请求体、响应、状态码
- 附带关键代码片段
```

## 2️⃣ CRISPE 框架

**适合创意类、探索类任务**

| 要素 | 含义 |
|------|------|
| **C**apacity | AI 扮演的角色/能力 |
| **R**equest | 具体任务请求 |
| **I**nspiration | 灵感来源/参考风格 |
| **S**teps | 执行步骤 |
| **P**ersonality | 输出的人格特征 |
| **E**xperiment | 尝试的不同方案 |

### 模板

```markdown
作为 [角色/专家]，请完成以下任务：

**任务**：[具体请求]

**参考风格**：[类似的作品/风格/示例]

**步骤**：
1. [第一步]
2. [第二步]
3. [第三步]

**输出风格**：[幽默/严肃/诗意/简洁等]

**请提供**：[2-3 个不同方案供选择]
```

## 3️⃣ BROKE 框架

**适合有约束条件的任务**

| 要素 | 含义 |
|------|------|
| **B**ackground | 背景信息 |
| **R**ole | AI 扮演的角色 |
| **O**bjectives | 目标列表 |
| **K**onstraints | 约束条件 |
| **E**xamples | 示例参考 |

### 模板

```markdown
**背景**：[任务背景]

**你的角色**：[专家角色]

**目标**：
- [目标 1]
- [目标 2]
- [目标 3]

**约束条件**：
- [限制 1：如字数、时间、技术等]
- [限制 2]
- [限制 3]

**参考示例**：
[提供 1-2 个理想输出的例子]
```

## 4️⃣ TAG 框架 ⚡ 快速任务

**适合简单、快速的任务**

| 要素 | 含义 |
|------|------|
| **T**ask | 任务是什么 |
| **A**ction | 需要执行的动作 |
| **G**oal | 期望达成的目标 |

### 模板

```markdown
**任务**：[一句话描述]
**动作**：[写/分析/总结/翻译/改写...]
**目标**：[期望的结果]
```

### 示例

```markdown
**任务**：这篇技术文章太长了
**动作**：总结核心要点
**目标**：让我 3 分钟内理解文章主旨
```

---

## 5️⃣ RTF 框架 📄 格式化输出

**适合需要特定格式的任务**

| 要素 | 含义 |
|------|------|
| **R**ole | 角色 |
| **T**ask | 任务 |
| **F**ormat | 输出格式 |

### 模板

```markdown
作为 [角色]，请 [任务]，输出格式为 [表格/JSON/Markdown/列表/代码等]
```

---

## 🎯 框架选择指南

| 场景 | 推荐框架 |
|------|----------|
| 复杂项目/技术任务 | **CO-STAR** |
| 创意写作/头脑风暴 | **CRISPE** |
| 有明确限制的任务 | **BROKE** |
| 简单快速任务 | **TAG** |
| 需要特定格式 | **RTF** |
| 日常通用 | **CO-STAR** (最稳妥) |

## 💡 进阶技巧

### 1. 框架组合使用

```markdown
# 使用 CO-STAR 定义整体结构
# 在其中嵌入 BROKE 的约束条件

## Context
...

## Objective
...

## Constraints (来自 BROKE)
- 不超过 500 字
- 使用 Python 3.10+
- 不依赖外部库
...
```

### 2. 添加"思维链"指令

在复杂任务前加上：
```markdown
请逐步思考，展示你的推理过程。
Let's think step by step.
```

### 3. 提供"反例"

```markdown
**不要这样做**：
[给出糟糕示例]

**要这样做**：
[给出优秀示例]
```

### 4. 设定评估标准

```markdown
**优秀回答的标准**：
- 准确性：信息正确无误
- 完整性：覆盖所有要点
- 实用性：可直接应用
- 清晰度：易于理解
```

---

*记住：框架是工具，不是枷锁。熟练后可以灵活组合或简化。*
